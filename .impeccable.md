## Design Context

### Users
ReqAgent 当前服务于产品经理、业务负责人和技术负责人这类高杠杆决策者。
他们通常在需求仍然模糊、需要快速形成结构化判断和执行方向时打开这个界面。
这个产品不是聊天玩具；它是一个帮助他们和 agent 协作、拆解问题、推进项目的工作台。

### Brand Personality
品牌气质应当是：克制、精准、可信。
界面需要传达“这是一个冷静而可靠的专业系统”，而不是炫技式 AI demo。
交互应当有细节和质感，但不喧宾夺主，不依赖情绪化视觉噱头。

### Aesthetic Direction
整体方向为 light-first 的精密工作台，偏向编辑性与操作台的结合体。
视觉基底保持黑白与暖灰体系，参考 OpenAI 的克制感，但不能复制其营销页面语言。
不使用渐变、不堆大卡片、不做泛 AI 风格的发光或玻璃效果。
允许极少量语义色，但只用于状态和反馈，而不是装饰。
动效应当轻、短、细，重点放在层级切换、折叠展开、状态反馈与输入响应上。

### Design Principles
1. Primary content stays readable. Keep the conversation as the main surface and compress process details after completion.
2. Minimal does not mean flat. Use rhythm, type, motion, and micro-contrast to create sophistication without visual noise.
3. Status must be legible at a glance. Tool execution, reasoning, loading, and failure states need distinct semantics and affordances.
4. Chinese-first means operational language too. User-visible process labels should feel native, not like developer placeholders.
5. The interface should feel directed, not decorative. Every visual accent must either improve hierarchy, signal state, or support action.
