# ReqAgent — 智能需求分析平台

## 项目汇报

---

## 一、我们要解决什么问题

当前需求分析过程中存在三个核心痛点：

**效率低** — PM 手工编写 PRD 耗时长，从需求沟通到文档输出通常需要 1-2 周。需求变更时文档修改成本高，经常出现版本不一致的问题。

**质量不稳定** — PRD 质量高度依赖个人经验。新人容易遗漏非功能需求、合规要求、边界条件。评审时反复返工，同样的问题在不同项目中重复出现。

**知识没有沉淀** — 历史项目的需求分析经验散落在个人电脑里，没有形成可复用的组织资产。新项目不能有效参考过往经验。

---

## 二、我们的方案

构建一个 **AI 驱动的需求分析助手**（ReqAgent），作为多 Agent 编排平台的第一个 Agent。

### 它能做什么

用户在 Web 界面上输入需求描述（文字、截图、Word 文档等），Agent 自动完成：

```
用户输入需求 → Agent 智能追问补全信息
             → 自动检索合规要求和历史参考
             → 输出结构化 User Story（含优先级和验收标准）
             → 生成完整的 PRD 文档（符合公司模板规范）
```

### 核心能力

| 能力 | 说明 |
|------|------|
| 智能追问 | 自动识别需求描述中的缺失信息，主动向用户追问关键问题 |
| 合规检查 | 自动关联金融合规要求（个保法、反洗钱、数据分类分级等） |
| 知识复用 | 检索历史项目 PRD，参考类似功能的拆分方式和常见问题 |
| 标准化输出 | 按公司 PRD 模板生成文档，确保格式和内容规范 |
| 领域适配 | 不同项目组可配置自己的业务知识和模板（Skill 热插拔） |

## 三、产品形态

### 用户交互流程

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ① 用户输入        ② Agent 分析         ③ 产出物       │
│                                                         │
│   一句话描述  ──→   智能追问      ──→   User Story 看板  │
│   需求文档    ──→   合规检查      ──→   PRD 文档         │
│   竞品截图    ──→   知识库检索    ──→   数据流图         │
│   会议纪要    ──→   需求拆解      ──→   优先级矩阵       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 界面布局

左侧为对话区域，用户与 Agent 自然语言交互；右侧为产出物面板，实时展示生成的 User Story 看板和需求文档。

Agent 执行过程中，界面会实时显示当前状态（"解析文档中"、"检索合规要求"、"生成 User Story"），让用户清楚了解进度。

## 四、Skill 热插拔 — 适配不同业务线

### 设计理念

Agent 本身是通用的需求分析引擎。不同项目组通过加载不同的 **Skill**（领域知识包）来适配自己的业务场景。

```
同一个 Agent 引擎
    │
    ├── 加载"银行零售 Skill"   → 变成银行零售需求分析专家
    ├── 加载"证券交易 Skill"   → 变成证券交易需求分析专家
    └── 加载"风控平台 Skill"   → 变成风控平台需求分析专家
```

### Skill 包含什么

| 组成 | 说明 | 示例 |
|------|------|------|
| 领域指令 | 告诉 Agent 该领域的分析重点 | "银行零售业务需关注渠道适配" |
| PRD 模板 | 该业务线的标准文档模板 | 公司内部 PRD 模板 |
| 合规知识 | 相关的监管和合规要求 | 个保法、反洗钱条款 |
| 历史参考 | 过往项目的 PRD 摘要 | 手机银行项目需求参考 |
| 领域术语 | 专业术语标准定义 | 头寸、T+0、KYC 等 |

### 谁来维护 Skill

**业务团队**直接维护，不需要写代码。Skill 就是一组 Markdown 文件，按约定格式整理即可。技术团队只负责 Agent 引擎和平台能力。

## 五、技术方案概要

### 架构选型

| 层级 | 选择 | 理由 |
|------|------|------|
| 整体框架 | 全栈 TypeScript (Next.js) | 一个项目、一个语言、部署简单 |
| Agent 引擎 | OpenAI Agents SDK | 轻量、支持多 Agent 编排、行业主流 |
| 工具层 | MCP 协议（行业标准） | 文档解析、文件操作等能力开箱即用 |
| 前端 UI | assistant-ui + Vercel AI SDK | 成熟的 AI 对话组件库 |

### 架构简图

```
┌──────────────────────────────────────────────┐
│  Web 界面                                     │
│  AI 对话组件 + 产出物展示面板                   │
└──────────────┬───────────────────────────────┘
               │
┌──────────────▼───────────────────────────────┐
│  Agent 引擎                                   │
│  需求解析 → 知识检索 → 需求拆解 → 文档生成     │
│  + Skill 知识注入（按项目组动态加载）           │
└──────────────┬───────────────────────────────┘
               │
┌──────────────▼───────────────────────────────┐
│  工具层                                       │
│  文档解析 · 知识库检索 · 合规检查 · 文档导出   │
└──────────────────────────────────────────────┘
```

## 六、多 Agent 平台演进路线

ReqAgent 是平台的第一个 Agent。架构设计已考虑后续扩展，新增 Agent 只需定义能力和知识，不需要改底层架构。

```
Phase 1（当前）        Phase 2            Phase 3          Phase 4
需求拆解 Agent    →   领域建模 Agent  →  编码 Agent   →  Review Agent
                      ER 图               脚手架生成       代码审查
User Story            API 设计            单元测试         安全扫描
PRD 文档              数据模型            代码生成         质量检查
```

每个 Agent 的产出物直接作为下一个 Agent 的输入，形成**从需求到代码的全自动化链路**。

## 七、开发计划

| 阶段 | 内容 | 周期 | 交付物 |
|------|------|------|--------|
| **Phase 1** | MVP — 文本对话 + User Story + PRD 生成 | 2 周 | 可演示的 Demo |
| **Phase 1.5** | Skill 热插拔验证 + 示例 Skill | 1 周 | Skill 机制 + 1 个业务 Skill |
| **Phase 2** | 文件上传 + 长文档处理 + 文档导出 | 2 周 | 完整的需求分析流程 |
| **Phase 3** | 真正的多 Agent 编排 + Pipeline 可视化 | 2-4 周 | 多 Agent 协作 |
| **Phase 4** | 新增领域建模/编码/Review Agent | 1-2 月 | 完整平台 |

### 关键里程碑

- **第 2 周末**：可演示 Demo（用户输入需求 → 自动输出 User Story + PRD）
- **第 5 周末**：支持文件上传、Skill 热插拔、文档导出
- **第 9 周末**：多 Agent 协作版本

## 八、需要的支持

### 技术侧（开发团队）
- 全栈 TS 开发 1-2 人
- OpenAI API Key（GPT-4o）

### 业务侧（最关键）
- **PM 团队**：提供公司标准 PRD 模板和撰写规范
- **合规团队**：提供金融监管要求摘要（个保法、反洗钱、数据分级等）
- **各项目组**：提供 2-3 份历史项目 PRD 作为参考
- **业务分析师**：提供领域术语标准定义

> 业务知识的质量直接决定 Agent 的产出质量。框架是管道，知识是水。

## 九、预期价值

| 维度 | 当前 | 使用 ReqAgent 后 |
|------|------|-----------------|
| PRD 编写周期 | 1-2 周 | 1-2 天（初稿） |
| 合规要求遗漏率 | 依赖个人经验 | 自动检查，接近零遗漏 |

| 知识复用 | 散落在个人电脑 | Skill 沉淀为组织资产 |
| 跨项目一致性 | 各项目组标准不一 | 统一模板 + Skill 保证一致 |
